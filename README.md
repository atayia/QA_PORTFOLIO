## How to Navigate This Portfolio

The portfolio is designed to be browsed as a website, not raw files.

1. Open `index.html` to reach the **landing page** — it links to all active client audits.
2. Each client folder has a **summary page** (e.g. `amazon/amazon.html`) with tiles linking to:
   - Test Plan
   - Requirements / RTM
   - Test Cases (with embedded screenshot evidence)
   - Bug Reports
   - Automation (placeholder or active)
3. All links use **relative paths** — safe to clone or rename the repo.
4. Markdown files (`.md`) are **source/reference artifacts** kept alongside HTML pages for transparency.
